# volss - Shadowsocks-Rust 一键管理脚本

> 版本: V1.0.9 · 快捷命令: `volss`

一个适用于 NAT 机/VPS 的 Shadowsocks-Rust 交互式安装与管理脚本，支持多用户管理、流量统计、ACL 黑名单等功能。

---

## 功能概览

- 一键安装/卸载最新版 Shadowsocks-Rust
- 交互式选择加密方式、端口分配策略、用户数量
- 顺序端口或随机端口分配，自动跳过已占用端口
- 多用户管理：暂停、恢复、删除、重新生成
- 基于 iptables 的每用户流量统计（GB）与重置
- ACL 黑名单：屏蔽指定域名，默认放行所有其他流量
- systemd 服务管理：启动、停止、重启、日志
- 一键更新脚本（从 GitHub 拉取最新版，自动备份旧版）
- 快捷命令 `volss` 随时呼出管理菜单

## 安装流程说明

选择安装后，脚本会依次引导完成以下配置：

### 第一步：选择加密方式

```
1) 2022-blake3-aes-128-gcm        (推荐，密钥16字节)
2) 2022-blake3-aes-256-gcm        (强加密，密钥32字节)
3) 2022-blake3-chacha20-poly1305   (ARM推荐，密钥32字节)
4) aes-256-gcm                    (传统，兼容性好)
5) chacha20-ietf-poly1305         (传统，兼容性好)
```

> 推荐使用 `2022-blake3-aes-128-gcm`，性能与安全性均衡。

### 第二步：填写服务器信息

输入服务器域名或 IP，留空则自动检测。支持 DDNS 域名（如 `hkt.289599.xyz`）。

### 第三步：端口分配

```
1) 顺序端口 —— 从指定起始端口开始，自动跳过已占用端口
2) 随机端口 —— 在指定范围内随机挑选可用端口
```

**顺序模式示例：**
```
起始端口: 30001
生成数量: 5

端口 30001 可用 ✓
端口 30002 已占用，跳过
端口 30003 可用 ✓
端口 30004 可用 ✓
端口 30005 可用 ✓
端口 30006 可用 ✓
```

**随机模式示例：**
```
端口范围: 20000 ~ 60000
生成数量: 5

端口 43521 已分配 ✓
端口 28974 已分配 ✓
...
```

### 第四步：配置 ACL 黑名单（可选）

输入需要屏蔽的域名，每行一个，空行结束。安装完成后也可随时通过菜单增删。

## 功能详解

### 用户管理

| 功能 | 说明 |
|------|------|
| 查看用户列表 | 显示所有用户的端口、加密方式、状态（正常/暂停） |
| 查看 SS 链接 | 输出所有 `ss://` 格式链接，可直接导入客户端 |
| 暂停用户 | 将指定用户从运行配置中移除，不删除数据 |
| 恢复用户 | 重新将暂停的用户加入运行配置 |
| 删除用户 | 永久删除用户配置、链接及对应 iptables 规则 |
| 重新生成所有用户 | 保留端口数量和加密方式，重新生成所有密码 |

### 更新脚本

选择菜单选项 `19) 更新脚本`：

- 从 GitHub 拉取最新版本
- 自动对比本地与远程版本号
- 版本相同则提示无需更新
- 发现新版本询问确认后更新
- 更新前自动备份当前脚本为 `.bak`
- 更新完成后自动重启进入新版菜单

基于 iptables 对每个端口单独计数，统计维度：

- **上行（GB）**：服务端发送给客户端的流量（用户下载）
- **下行（GB）**：客户端发送给服务端的流量（用户上传）

> **注意：统计数值为单向流量。** 由于代理服务器每个字节都要经过一进一出，VPS 实际消耗的带宽约为统计数值的 **2 倍**。例如显示上行 21GB，实际带宽消耗约为 42GB。流量数据在服务器重启后重置，如需持久化建议定期记录。

### ACL 黑名单

通过 Shadowsocks-Rust 的 ACL 功能屏蔽指定域名，被屏蔽的域名请求将被直接拒绝。

常见使用场景：
- 屏蔽 IP 查询类网站（防止用户检测真实 IP）
- 屏蔽广告域名
- 屏蔽特定服务

## 常见问题

**Q: 菜单显示"未运行"但连接正常？**

部分 LXC 容器宿主机会定期向容器内进程发送 SIGTERM 信号（约每 54 秒一次），导致 ssserver 被杀后立即重启。由于 systemd 状态在重启间隙短暂变为非 active，旧版脚本可能误判为"未运行"。V1.0.6 起改用 `pgrep` 检测进程是否实际存在，不再依赖 systemctl 状态，显示更准确。如果频繁断连影响使用，建议联系服务商关闭该限制。

**Q: 如何更新脚本？**

进入菜单选择 `19) 更新脚本`，脚本会自动从 GitHub 拉取最新版本，对比版本号后询问确认，更新前自动备份当前版本为 `.bak` 文件，完成后自动重启进入新版菜单。

**Q: NAT 机如何使用？**

NAT 机通常只开放有限端口，填写起始端口时请使用服务商分配的可用端口段。服务器地址填写服务商提供的公网 IP 或 DDNS 域名。

**Q: 重启服务器后流量数据丢失？**

iptables 规则在安装时已通过 `netfilter-persistent` 持久化，规则本身不丢失，但计数器会清零。流量统计从上次规则加载后开始重新累计。

**Q: 暂停和删除有什么区别？**

暂停仅将用户从运行配置中移除，数据保留，可随时恢复。删除会永久移除用户的所有配置和链接，不可恢复。

**Q: 支持重新安装吗？**

支持。选择菜单选项 1 时，若检测到已安装会询问是否重新安装，确认后走完整安装流程。
